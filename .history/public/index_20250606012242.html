1. Basics of Java

1.1. What is Java? Java is a high-level, class-based, object-oriented programming language that is designed to have as few implementation dependencies as possible. It follows the ‚ÄúWrite Once, Run Anywhere‚Äù (WORA) philosophy.

1.2. Java Virtual Machine (JVM): The JVM is the runtime environment that executes bytecode. It provides platform independence by abstracting away hardware/OS specifics.

1.3. JDK vs. JRE:

  • JDK (Java Development Kit): Contains compilers (javac), tools, and libraries to develop Java applications.
  • JRE (Java Runtime Environment): Contains the JVM and core libraries needed only to run Java programs (no compiler).

2. Data Types & Variables

2.1. Primitive Data Types:

  • byte: 8-bit signed integer
  • short: 16-bit signed integer
  • int: 32-bit signed integer
  • long: 64-bit signed integer
  • float: 32-bit floating-point
  • double: 64-bit floating-point
  • char: 16-bit Unicode character
  • boolean: true or false

2.2. Reference Types: Objects, arrays, and interfaces. Stored as references to memory addresses.

2.3. Variable Declaration: int count = 10; By default, uninitialized instance variables get default values (0 for numerics, false for boolean, null for references).

3. Operators & Expressions

3.1. Arithmetic Operators: +, -, *, /, %

3.2. Relational Operators: ==, !=, >, <, >=, <=

3.3. Logical Operators: && (AND), || (OR), ! (NOT)

3.4. Assignment Operators: =, +=, -=, *=, /=, %=

3.5. Increment/Decrement: ++i, i++, --i, i--

3.6. Ternary Operator: condition ? exprIfTrue : exprIfFalse

4. Control Flow Statements

4.1. if / else if / else:


if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else {
    grade = 'C';
}

4.2. switch:


switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    default:
        System.out.println("Weekend");
}

4.3. Loops:

  • for: for (int i = 0; i < 5; i++) { ... }
  • while: while (condition) { ... }
  • do-while: do { ... } while (condition);

4.4. for-each Loop (Enhanced for): for (String item : itemList) { ... }

5. Arrays & Strings

5.1. One-Dimensional Array: int[] nums = new int[5]; or int[] nums = {1, 2, 3};

5.2. Multi-Dimensional Array: int[][] matrix = new int[3][3];

5.3. String Basics: Strings are immutable. String s = "Hello"; Common methods: length(), charAt(), substring(), indexOf(), equals(), toLowerCase()

5.4. StringBuilder / StringBuffer: Mutable alternatives to String for efficient concatenation. StringBuilder sb = new StringBuilder("Hi"); sb.append(" there");

6. Object-Oriented Programming (OOP)

6.1. Class & Object: - Class: Blueprint or template. - Object: An instance of a class. public class Car { String model; int year; void drive() { System.out.println("Driving"); } }

6.2. Constructors: Special method to initialize objects. public Car(String m, int y) { model = m; year = y; }

6.3. this Keyword: Refers to the current object. Useful when parameter names match field names.

6.4. Inheritance: Use extends to inherit: public class Vehicle { ... } public class Car extends Vehicle { ... }

6.5. Polymorphism: - Compile-time (Overloading): Same method name, different parameters. - Runtime (Overriding): Subclass provides specific implementation of a superclass method.

6.6. Abstraction: - Abstract Class: Can have both abstract and concrete methods. abstract class Animal { abstract void sound(); } - Interface: 100% abstract (until Java 8 default/static methods). interface Flyable { void fly(); }

6.7. Encapsulation: Keep fields private and provide public getters/setters. private int age; public int getAge() { return age; } public void setAge(int a) { if (a > 0) age = a; }

7. Exception Handling

7.1. try / catch / finally: try { int result = 10 / 0; } catch (ArithmeticException e) { System.out.println("Cannot divide by zero"); } finally { System.out.println("Always executes"); }

7.2. throw / throws: Use throw to explicitly throw an exception. Use throws in method signature to delegate exception handling. void readFile() throws IOException { ... }

7.3. Custom Exceptions: Extend Exception or RuntimeException to create your own. class MyException extends Exception { ... }

8. Collections Framework

8.1. List: Ordered, can contain duplicates. ArrayList, LinkedList

8.2. Set: No duplicates, unordered (unless you use SortedSet). HashSet, TreeSet, LinkedHashSet

8.3. Map: Key-value pairs, keys must be unique. HashMap, TreeMap, LinkedHashMap

8.4. Queue / Deque: LinkedList, ArrayDeque, PriorityQueue

8.5. Iterators: Used to traverse collections safely. Iterator it = list.iterator(); while (it.hasNext()) { System.out.println(it.next()); }

9. Threads & Concurrency

9.1. Creating Threads: - Extend Thread class and override run(). - Implement Runnable interface and pass to a Thread object. Thread t = new Thread(new MyRunnable()); t.start();

9.2. Synchronization: Use synchronized keyword to avoid race conditions. synchronized void increment() { count++; }

9.3. volatile Keyword: Ensures visibility of changes across threads. private volatile int flag;

9.4. Executor Framework (Java 5+): Use ExecutorService to manage thread pools. ExecutorService exec = Executors.newFixedThreadPool(5); exec.submit(() -> { /* task */ });

10. File I/O (java.io & java.nio Packages)

10.1. java.io (Stream-based):

  • FileReader / FileWriter: Character streams.
  • FileInputStream / FileOutputStream: Byte streams.
  • BufferedReader / BufferedWriter: Wrap FileReader/FileWriter for efficiency.

10.2. java.nio (New I/O, buffer-based):

  • Paths & Files: Path path = Paths.get("file.txt"); Files.readAllLines(path);
  • ByteBuffer: For reading/writing binary data.

11. JDBC (Java Database Connectivity)

11.1. Loading Driver: Class.forName("com.mysql.cj.jdbc.Driver");

11.2. Establish Connection: Connection con = DriverManager.getConnection(url, user, pass);

11.3. Executing Queries: Statement stmt = con.createStatement(); ResultSet rs = stmt.executeQuery("SELECT * FROM users"); while (rs.next()) { System.out.println(rs.getString("name")); }

11.4. PreparedStatement: Use placeholders to prevent SQL injection. PreparedStatement ps = con.prepareStatement("INSERT INTO users(name, age) VALUES(?, ?)"); ps.setString(1, "Alice"); ps.setInt(2, 25); ps.executeUpdate();

12. Java 8+ Key Features

12.1. Lambda Expressions: Shorter syntax for anonymous functions. List<String> names = Arrays.asList("Bob", "Alice"); names.forEach(name -> System.out.println(name));

12.2. Streams API: Functional-style operations on collections. List<Integer> nums = Arrays.asList(1,2,3,4,5); List<Integer> evens = nums.stream() .filter(n -> n % 2 == 0) .collect(Collectors.toList());

12.3. Optional: Prevents NullPointerException. Optional<String> opt = Optional.ofNullable(getName()); opt.ifPresent(System.out::println);

12.4. Default & Static Methods in Interfaces: Allows concrete methods in interfaces. interface Vehicle { default void start() { System.out.println("Starting"); } static void info() { System.out.println("Vehicle Info"); } }

13. Miscellaneous Topics

13.1. Annotations: Metadata for classes, methods, fields. Built-in: @Override, @Deprecated, @SuppressWarnings. Custom annotations require @interface.

13.2. Generics: Type-safe collections. List<String> list = new ArrayList<>();

13.3. Enumerations (enum): enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

13.4. Serialization: Convert objects to byte streams. Implement Serializable interface. FileOutputStream fos = new FileOutputStream("data.ser"); ObjectOutputStream oos = new ObjectOutputStream(fos); oos.writeObject(myObject);

13.5. Java Memory Model: - Heap: Stores objects. - Stack: Stores local variables and method calls. - Garbage Collection: Automatic memory management.
